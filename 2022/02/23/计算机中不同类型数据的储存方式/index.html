计算机中不同类型数据的储存方式

在计算机中各种类型的数据最终都是以二进制的方式储存的，称为“机器数”。而数据的真实值可能存在正负号，小数点等信息，所以要采用相应的编码方式将真实值转换为二进制的形式。常用编码方式有原码、补码、反码、移码

数据可分为定点数和浮点数，定点数分为定点小数（纯小数，整数位为0）与定点整数（无小数位），定点数的编码方式相对简单，而浮点数既有整数部分与小数部分，编码相对复杂。

编码前需要先将真值转化为对应的二进制真值。（还带有小数点正负号等信息）

定点机器数的表示方法

原码表示法

​ 原码表示是最接近真实值的表示方法。

​ 对于n位定点整数（小数点隐藏在最后一位后面），只需将它的符号进行编码，若为正，编码为0；负则编码为1，然后放在最前面。（加了一位符号位）

​ 对于定点小数（小数点隐藏在第一位后面），整数位（第一位）一定是0。对符号进行编码，然后对原来的整数位（一定是0，没有存储的意义）进行修改。说法“n位的二进制小数X”,不包括整数位的0，即小数部分有n位。

原码表示法的特点

​ 对于整数0，+0 与 -0 有两种不同的表示形式：【+0】的原码为00…00,【-0】的原码为10…00。占用了两个二进制编码。

​ 表示的真值的范围：

​ 定点整数：**-（2^n-1）<= X <= 2^n-1**

​ 定点小数：**-（1-2^(-n)）<= X<= 1- 2^(-n)**

​ 参与运算时，要单独考虑符号位，比较麻烦。补码的运用很好地解决了这一问题。

补码表示法

​ 对于正数的编码方式与原码的编码方式相同。而对于负数的编码，可以从运算的角度来理解。对于正数X, 0 = X + (-X),假设X的位数为 n 位，要使得加法运算后 n 位二进数全部变为0，可以使得两数相加产生一个最高位的进位（第n+1位），将它截断舍去，然后剩下的位数全部变成0。 根据这个思路，那么将负数的真值进行编码的算法是：设n位二进制负数 -X ，则可以用一个 n+1 位的二进制数：第一位为1，剩下n位全为0来减去X 。 而 100….0 - X = (100….0 - 1) - X + 1 = 【11…11 (n个1) - X】+1，【】中的操作相当于对X中的每一位取反，所以简而言之，负数的补码等于它的绝对值每一位取反后加“1”，加上符号位“1”。

​ 如果是将补码转化为原码，正数不变，负数则是上述过程的逆过程（不看符号位），即先减1 ，再每位取反。 考虑这个过程，即 11…11 (n位)- (X - 1) = 【11…11 - X】 + 1, 也是可以先取反再加1，再加上符号位1。

补码表示法的特点

​ 真值 +0 的表示方法唯一，即全是0。原码中原本表示-0的编码腾出来表示负数的最小值，即-2^n.

​ 补码的运算符号位和数值位一起参与。

反码表示法

​ 反码通常用于转换中的过度。

​ 正数的反码与原码、补码相同，而负数的反码是将所有位取反，符号位为1. 与补码的不同是取反后没有加1。

反码表示法的特点

​ 0 占用了两个编码。

​ 表示范围与原码相同

移码表示法

​ 移码主要用于浮点机器数的阶码，一般用来表示定点整数。 求移码只需要将补码的符号位取反即可。

浮点机器数的表示法

IEEE 754浮点数标准格式
类型 总位数 尾数尾数（包含一位数符） 阶码位数（包含一位阶符）
短实数 32 24 8
长实数 64 53 11
临时实数 80 65 15

浮点数的表示标准可能不同，要根据不同的要求。要先化为科学计数法的表示形式，尾数（定点小数） * 2^阶码（定点整数）次，如阶码用移码表示，偏移量是 2^n-1，而不是移码中的 2^n,尾数储存时隐藏最高位的1，使得尾数的精读更高。

字符编码

​ 用7位二进制数表示 127个ASC字符

汉字编码

​ 汉字输入—— 输入码

​ 汉字储存与交换——内码，交换码

​ 汉字输出——字形码

输入码也称外码，是为了将汉字输入计算机而编制的代码。

交换码是不同计算机系统之间交换汉字信息时使用的代码标准。

汉字内码：每个汉字内码是唯一的。英文字符用一个字节就能表示，且最高位为0. 一个汉字需要用两个字节表示，每个字节最高位均为1.

字形码根据点阵输出，简易型汉字的大小为16*16的点阵来表示，需要16*16/8 = 32个字节来表示。
